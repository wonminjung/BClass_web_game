## 3. 서버 로직 분석

### AuthService.ts

- In-memory Map 저장소 (DB 미사용)
- HMAC-SHA256 토큰 생성 (24시간 TTL)
- UUID 기반 세이브코드 자동생성
- 초기 인벤토리: HP포션x3, MP포션x1, 금화 100

### CombatService.ts (519줄, 핵심 로직)

- 플레이어 능력치: `baseStats + (레벨-1) * 스케일`
- 피해 공식: `(공격 * 배수 - 방어 * 0.5) * 크리보정`, 최소 1
- 크리티컬: baseCritRate 기반 랜덤
- 적 스킬 선택: 가중치 기반 랜덤
- 상태효과: poison/burn/bleed(지속피해), regen(HP회복), stun(행동불가)
- **미적용**: `defense_up`, `attack_up`, `shield` (데이터만 저장)

### GameService.ts

- 경험치: 누적 → `exp >= level * 100` 시 자동 레벨업
- 아이템: 스택 가능 확인 → 기존 슬롯 추가 또는 새 슬롯 생성
- 도감: 최초 처치일시 기록, 재처치 시 killCount 증가
- **미구현**: `useItem()`, `equipItem()` 함수 없음

## 4. 클라이언트 분석

### Zustand 스토어

- **authStore**: 인증/세이브데이터, axios 헤더 자동 설정
- **gameStore**: 캐릭터/던전 데이터 (로컬 import, 서버 API 미활용)
- **combatStore**: 전투 상태, 스킬 상태, 전투 로그, 보상/레벨업

### 커스텀 훅

- **useAuth**: 로그인/로그아웃/세이브 데이터 관리
- **useCombat**: 전투 진행, 스킬 사용 가능 여부 헬퍼 포함
- **useInventory**: 아이템 목록/사용/장비 (API 미구현 상태)

### 라우트 구조

```
/ → AuthScreen (로그인)
/select → CharacterSelect (캐릭터 선택)
/home → HomeScreen (메인 화면)
/dungeon → DungeonSelect (던전 선택)
/battle/:dungeonId → BattleScreen (전투)
/inventory → InventoryScreen (인벤토리)
/bestiary → BestiaryScreen (도감)
```

### 컴포넌트 특징

- React.memo, useCallback, useMemo 적극 활용
- 공통 컴포넌트 분리 (Button, Card, Modal, StatBar)
- SVG 아이콘 + onError 이모지 폴백 (feature/item-icons)
